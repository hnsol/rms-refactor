# Wiki的リンク・ナビゲーション運用ルール

## 目的
- RMSを、読み切り記事の集まりではなく、相互参照できる教科書として育てる。
- 読者が迷ったときに、キーワードや前後ナビゲーションから関連ページへ移動できるようにする。
- リンクを増やしすぎて本文の流れを壊さない。

## キーワードリンク
- 初出の重要キーワードは、関連する代表ページへリンクする。
- 同じページ内で同じキーワードを何度もリンクしない。
- 原則として、リンクするのは初出または重要箇所だけにする。
- キーワードは `rules/glossary.yml` にある語を優先する。
- 代表ページが未作成の場合は、無理に空リンクを増やさない。ページを作る予定が明確なときだけリンクする。

## フェーズとテクニックのリンク
- フェーズ記事から、関連するテクニック記事へリンクする。
- テクニック記事から、関連するフェーズ記事へ戻れるようにする。
- フェーズ記事ではテクニックの詳細を書きすぎない。詳細はテクニック記事に逃がす。
- テクニック記事では、どのフェーズで使うのかを明記する。

## ページ末尾ナビゲーション
- docs 配下の本文ページには、可能な範囲で末尾に軽いナビゲーションを置く。
- ナビゲーションは本文の補助であり、長くしすぎない。

```markdown
## 次に読む

- 前に読む: [前のページ](previous.md)
- 次に読む: [次のページ](next.md)
- 関連するテクニック: [ラポール形成](../techniques/rapport.md)
```

## リンク量の目安
- 1段落に複数リンクを詰め込まない。
- 1ページ内のリンクは、読み筋を助けるものに絞る。
- リンクは「迷った読者の避難路」として使う。
- 本文の主張よりリンクの存在感が強くなったら減らす。

## リンク切れへの対応
- `mkdocs build` で警告が出たリンクは、作業範囲内で直す。
- 作業範囲外の未作成ページが原因の場合は、無理に本文を書かず、必要に応じて scaffold ページを作る。
- ナビゲーションに載せるページは、実ファイルが存在する状態を目指す。
