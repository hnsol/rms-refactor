# 文体・構成ルール（ドラフト）

## 語り口
- 池上彰風の「わかる言葉で」「順序立てて」「軽妙に」。
- 結論先出し→理由→手当て→落とし穴→次へ。
- 短い問いかけを挟み、テンポを作る。

## 文章ルール
- 1段落は3〜5文まで。短く、判定可能に。
- 抽象語だけで押し切らず、現場のリアル→理由→手当てで説明する。
- 強調（太字）は最小限。

## docs 配下 Markdown ガイドライン
### リスト
- 行頭は半角ハイフン`-`を使用。  
- `-`と本文の間には**必ず半角スペース1文字**を入れる。  
- ネストは半角スペース2文字ずつインデントする。  
  - 正:  
    ```markdown
    - parent
      - child
    ```  
  - 誤:  
    ```markdown
    -  parent   # スペース2つ
    -child      # スペースなし
    ```  

### 見出し
- トップレベルは`##`から開始し、以降1段階下げる。  
  - 正:  
    ```markdown
    ## 見出しレベル1
    ### 見出しレベル2
    ```  
- 文頭と`#`の間は空けない。  

### 強調
- **必要最小限**の場合のみ`**太字**`を使う。  
- 斜体・装飾ラインは使用しない。  

#### 強調と全角記号（重要）
- **全角記号を強調範囲に含めない**  
  - 該当記号（例示）: 「 」 『 』 （ ） 【 】 、 。 ・ ！ ？ ― … 〜 等  
  - 書式:  
    - 正: `「**展開の想定が圧縮の質を規定する**」`  
    - 誤: `**「展開の想定が圧縮の質を規定する」**`  
- strongタグ・ゼロ幅スペース・CSSハックは使用しない。  
- プレビューで太字が記号まで伸びていないか必ず確認。  

## フェーズ章の最低構成（軽量）
- フェーズの定義（1〜2文）
- 必要スキル（3つ目安）
- 代表テクニック（リンク）
- ここでやること（箇条書き）
- 落とし穴（誤解1つ＋失敗1つ）
- 次へ（次フェーズ＋関連テクニック）

## positioning.md から抽出した作法
- 導入で「うまくいかない現場」を描写する。
- 「話のなかみ以前の問題」を示す。
- 比喩は1回まで（礼法・儀式・境界など）。
- 技法は「媚びではない」「交通整理」など誤解を外す一言を添える。
