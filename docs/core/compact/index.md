## RMSの6つのスキル

| スキル           | 補記       | 要点                               |
| ---------------- | ---------- | ---------------------------------- |
| ポジショニング   | 目線あわせ | 味方として立ち位置と目的をそろえる |
| ディスカバリング | 真因特定   | 現象の裏の真因を掘り当てる         |
| ビルディング     | 認識あわせ | 必要性と優先順位の腹落ちを作る     |
| プレゼンティング | 解決策並べ | 選択肢比較→動ける計画に落とす      |
| セキュアリング   | 行動の合意 | 何を・誰が・いつまでにを合意       |
| リゾルビング     | 懸念ほぐし | 不安を言語化し、詰まりをほぐす     |


## 基本版

6つのスキルを「ひとことで・誤解・失敗」でまとめました。詳しい内容は [本編 > 基本版](core/standard/index.md) を確認ください。

## ポジショニング（目線あわせ）

- ひとことで: 相手にとって自分が「目標をともに達成する味方」と認識してもらうスキル
- よくある誤解: いちど行えば終わりと思ってしまう
- よくある失敗: 目的を伝えないまま本題に入り、相手を身構えさせてしまう
- フレーズ例
    - 1on1: スムーズに成果を出せるよう、今日は一緒に考えたいんだ
    - 会議: 今日の目的は〇〇を決めることです。進めてもよいでしょうか？
    - 家庭: 怒るんじゃなくて、楽しく片付けられる方法を相談したいんだ

## ディスカバリング（真因特定）

- ひとことで: 対話を通じ、現象の裏にねむる本当の原因を一緒に突き止めるスキル
- よくある誤解: プレゼンティングなど他のステップが混じる
- よくある失敗: 押し付けるか、聴きすぎるか、両極端になる
- フレーズ例
    - 1on1: どのプロセスで一番時間がかかっていると感じているかな？
    - 会議: 起きたトラブルの背景には、他にどんな要因が隠れていそう？
    - 家庭: 何がいちばん難しい？どこで手が止まりやすいか教えて？

## ビルディング（認識合わせ）

- ひとことで: 真因に対処する必要性の高さの認識を合わせるスキル
- よくある誤解: 理屈だけで相手を論破し、無理やり納得させることだと思ってしまう
- よくある失敗: 相手が必要性を実感しないまま、解決策検討に進んでしまう
- フレーズ例
     - 1on1: もしこのままの状況が続いたら、どんな影響が出ると思う？
     - 会議: 問題を解決できたら、いくらのコストメリットがあるでしょう？
      - 家庭: いま終わらせたら、夜はどれくらい遊ぶ時間があるかな？

## プレゼンティング（解決策並べ）

- ひとことで: 共に決め、みんなが納得して動ける実行計画を立てるスキル
- よくある誤解: よさそうな解決策があれば、それを選べばよい
- よくある失敗: 実行段階で迷い、柔軟な修正ができない
- フレーズ例
    - 1on1: どうすれば解決できると思う？アイデアを聞かせてほしい
    - 会議: 複数の案が出ましたが、それぞれのメリット・デメリットは？
    - 家庭: どうすれば忘れ物を防げると思う？良い方法を一緒に考えよう

## セキュアリング（行動の合意）

- ひとことで: 何を誰がいつまでにやるかを約束するスキル
- よくある誤解: RMS的には決めるところまで。実行確認は別ステップ
- よくある失敗: タスクが大きすぎ、誰が・いつまでにが曖昧になる
- フレーズ例
    - 1on1: 来週の火曜までにやる、で間違いないかな？
    - 会議: まずは〇〇から着手、でいいでしょうか？
    - 家庭: 明日の朝までに準備する内容を、もう一度おさらいしておこうか

## リゾルビング（懸念ほぐし）

- ひとことで: 不安や疑問をチャンスと捉え、心のブレーキを外すスキル
- よくある誤解: 他の5つのスキルで解決すべきこともリゾルビングに含める
- よくある失敗: 小さな不安を「大丈夫」と流し、不信感を残す
- フレーズ例
    - 1on1: ほかに何か心配なことは残っていないかな？
    - 会議: 手順について疑問や懸念があれば何でも言ってほしい
    - 家庭: 何かイヤだ、面倒だと思うことはある？正直に言って大丈夫だよ
