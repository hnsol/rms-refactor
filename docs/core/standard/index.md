## RMSの大目的

会議が進まない。合意したはずのことが動かない。そんな現場、思い当たりますか？よくあるのは「言ったのに動かない」「わかっているのに続かない」という停滞です。

RMS（Relationship Management Skills）は、そうした停滞をほどき、チームが前に進むための「共通の作法」を整理した体系です。対象は会社だけではありません。家庭や小さなコミュニティでも、人が一緒に動くならそこは組織です。

マネジメントは、思い通りに動かすことではありません。むしろ、各人が安心して力を出せる「ごきげんなチーム作り」です。しんどさはあっても、前進の手応えがあり、関係がこじれにくい状態ですね。では、どうやってそこにたどり着くのでしょうか。

答えはシンプルで、合意と実行が回る状態をつくることです。そのために、RMSは関係性の基盤を整え、チームが前に進むための「共通の作法」を用意します。

大規模な調査でも、心理的安全性が高いほどチームがのびのび活動し、成果につながりやすいと示唆されています。だからこそ、RMSは「合意を回す作法」と同時に、「安心して意見を出せる空気」を重視します。

この大目的を支える具体的な柱として、RMSには3つのサブ目的があります。
- マネジャーが役割を果たす
- 問題を解決する
- 支配ではなく協働する
がそうです。詳しくは [RMSの目指す世界](purpose/index.md) で確認してください。

このあと、RMSの流れを2本立てで案内します。ひとつは、目的の軸で「目的の束」を整理する話。もうひとつは、実行の軸で「フェーズ」を順にたどる話です。どちらも別々ではなく、行き来しながら理解を深めるための道案内だと思ってください。
