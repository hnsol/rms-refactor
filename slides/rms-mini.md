# RMSミニ全体図（問題解決モデル）

- 流れ: ポジショニング → ディスカバリング → ビルディング → プレゼンティング → セキュアリング → リゾルビング
- ポジショニング: 問いの入口を整え、解くべき領域と尺度を決める
- ディスカバリング: 事実と仮説を往復し、真因を一点に絞る
- ビルディング: 関係者の理解を揃え、合意できる前提を築く
- プレゼンティング: 解決案を運用可能な手順と判断材料に変換する
- セキュアリング: 次の一手を確約させ、実行に結びつける
- リゾルビング: 全工程を統合し、収束と後処理まで完遂する
- ハード/ソフト比率: ソフト寄り=ポジショニング/ビルディング/セキュアリング、ハード寄り=ディスカバリング/プレゼンティング、総合=リゾルビング

---

# ポジショニング（位置付け）

- 一言定義: 問題の輪郭を描き、取り組みの「座標」を決める技術
- よくある誤解: 立場表明だけがポジショニングだと思う
- よくある失敗: 目的が曖昧なまま議論を始める
- ケース別ミニ事例: 会議=議題の範囲を3分で固定 / 1on1=本人の悩みを目的文に変換 / プライベート=旅行の優先順位を先に宣言
- 使うフレーズ例: 「今回は○○までを対象にします」 / 「この議論のゴールは△△です」

---

# ディスカバリング（真因特定）

- 一言定義: 事実の裏にある因果を掘り当て、核心を見つける技術
- よくある誤解: 目立つ症状を真因だと決めつける
- よくある失敗: データ不足のまま結論に飛びつく
- ケース別ミニ事例: 会議=数字の異常値の理由を5回掘る / 1on1=モヤモヤの起点を具体場面に落とす / プライベート=家事が回らない原因を生活動線で確認
- 使うフレーズ例: 「なぜそれが起きたと言えますか？」 / 「根拠となる事実はどこですか？」

---

# ビルディング（認識合わせ）

- 一言定義: 関係者の前提を揃え、同じ地図を持つ技術
- よくある誤解: 意見の一致さえあれば十分だと思う
- よくある失敗: 用語や前提のズレを放置する
- ケース別ミニ事例: 会議=用語の定義を最初に確認 / 1on1=期待値の基準を共有 / プライベート=家族ルールの意味をすり合わせ
- 使うフレーズ例: 「ここでの○○は△△という理解で合っていますか？」 / 「前提を一度揃えてから進めます」

---

# プレゼンティング（実行プランの検討）

- 一言定義: 解決策を行動計画に落とし、選択可能な形にする技術
- よくある誤解: スライドを整えれば説得できると思う
- よくある失敗: 実行条件やリスクを提示しない
- ケース別ミニ事例: 会議=案ごとに工数と効果を並べる / 1on1=次の1週間の試行を提案 / プライベート=家計改善の手順を3段階で説明
- 使うフレーズ例: 「実行手順はA→B→Cです」 / 「判断材料はこの3点です」

---

# セキュアリング（行動の確約）

- 一言定義: 合意を行動に変え、責任と期限を確定する技術
- よくある誤解: 口頭の同意がそのまま実行になると思う
- よくある失敗: 役割と期限を曖昧にしたまま終える
- ケース別ミニ事例: 会議=担当者と期限を最後に復唱 / 1on1=次回までの約束を具体化 / プライベート=週末の予定をカレンダーに即入れる
- 使うフレーズ例: 「では○○さんが△日までに進めます」 / 「この一歩を約束できますか？」

---

# リゾルビング（収束・後処理）

- 一言定義: 全工程を束ねて決着させ、学びまで回収する総合技
- よくある誤解: 決定した瞬間に完了だと思う
- よくある失敗: 振り返りやフォローを省いて再発する
- ケース別ミニ事例: 会議=決定事項と次回確認点を整理 / 1on1=実行後の振り返り時間を設定 / プライベート=やった結果を共有して改善点を残す
- 使うフレーズ例: 「ここまでの結論と次の確認はこれです」 / 「実行後に必ず振り返りましょう」
